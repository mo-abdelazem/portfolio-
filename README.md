# Mohamed Abdelazem — Portfolio & Blog

A bilingual (English / Arabic) developer portfolio and technical blog, built with the Next.js App Router. The blog doubles as a structured **learning-path** platform: standalone articles plus curated curricula that take a reader from fundamentals to job-ready, one topic at a time.

🌐 [mohamed.work](https://mohamed.work)

## Features

- **Internationalization** — English & Arabic via [next-intl](https://next-intl.dev), with full RTL support, localized metadata, and `hreflang` alternates. Default locale (`en`) is unprefixed; Arabic lives under `/ar`.
- **Content-driven blog** — MDX powered by [Velite](https://velite.js.org). Metadata and content are split, so listings, the sitemap, and `generateStaticParams` only read frontmatter; the article body compiles per post page.
- **Learning paths** — a curriculum (categories → sections → ordered topics) defined once in the message files. Posts attach to a topic by their `translationKey`; unwritten topics render as "coming soon".
- **Tags & discovery** — tag pages, a sidebar of paths + tags, scroll-spy table of contents, reading time, and prev/next that walks the learning path.
- **Dark mode** — `data-theme` toggle with a no-flash inline script that respects (and live-follows) the device's `prefers-color-scheme`.
- **Modern UI** — an iOS-style glass pill navbar, a vertical section navigator with scroll-spy, animated reveals, a reading-progress bar, and theme-aware SVG diagrams — all RTL-aware.
- **SEO** — per-route canonical/alternates, OpenGraph/Twitter metadata, JSON-LD, dynamic social images, sitemap, and robots.

## Getting started

```bash
npm install
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

> **Note:** content is generated by Velite into `.velite/` (gitignored). The npm scripts run Velite for you, but a bare `next build`, `tsc`, or `eslint` will fail on the `#site/content` import until Velite has run at least once — run `npx velite` first if you invoke those directly.

### Scripts

| Script | Description |
| --- | --- |
| `npm run dev` | Build content, then start the dev server |
| `npm run build` | Build content, then build for production |
| `npm run start` | Start the production server |
| `npm run content` | `velite --watch` — rebuild content on change (run alongside `dev` while authoring) |
| `npm run lint` | ESLint |

## Project structure

```
messages/                 # en.json / ar.json — UI strings, site content, learning-path curriculum
src/
  app/[locale]/           # localized routes
    blog/                 # /blog (home), [slug], learn/[category], tag/[tag]
  components/             # nav, theme toggle, blog/section components, MDX renderer
  content/blog/{en,ar}/   # MDX posts (one file per locale)
  i18n/                   # next-intl routing / navigation / request config
  lib/                    # blog data (Velite), curriculum, types
  proxy.ts                # next-intl middleware (Next 16 "proxy" convention)
velite.config.ts          # content schema (frontmatter, TOC, reading time, compiled MDX)
```

## Authoring content

### Add a blog post

Create an MDX file per locale (same filename) under `src/content/blog/<locale>/`:

```mdx
---
title: "Your Post Title"
description: "One-line summary used on cards and in metadata."
publishedAt: "2026-06-10"
translationKey: "your-post-slug"
tags: ["CSS", "Layout"]
---

Your content. Every `##` / `###` heading joins the table of contents.
```

The filename becomes the slug (`/blog/your-post-slug`). Reading time is computed automatically — do **not** add it to frontmatter. A standalone post appears on `/blog` and its tag pages with no further wiring.

### Add it to a learning path

Edit `Roadmap.categories` in **both** `messages/en.json` and `messages/ar.json` (keep the `key`s identical across locales). A post lights up its topic automatically when its `translationKey` matches a topic `key`:

```jsonc
{ "key": "your-post-slug", "title": "Your Topic" }
```

### Remove a post

Delete its MDX files (both locales) and its topic entry from the curriculum, then rebuild.
